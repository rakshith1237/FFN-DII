# FFN Public API — OpenAPI 3.0 Specification
**Document:** 27_FFN_Public_API_OpenAPI.md
**Version:** 1.0.0
**Base URL:** https://hirenowwithflex.us/api/v1
**Authentication:** X-API-Key header
**Rate Limit:** 100 requests per minute per key (sliding window)

## Authentication

All requests must include the X-API-Key header.
API keys are generated by Partner Super Admins at /partner/settings/api-keys.
Keys are shown once on creation — store them in your secrets manager immediately.

## Error Responses

| Status | Meaning |
|---|---|
| 401 | Invalid or missing API key |
| 403 | Insufficient scope for this operation |
| 409 | Resource already exists (duplicate email) |
| 429 | Rate limit exceeded |
| 500 | Internal server error |

## GET /v1/candidates

List candidates for your organisation.
Required scope: read

Query parameters: page (integer, default 1), limit (integer, default 20, max 100)

Response 200: { data: [...candidates], pagination: { page, limit, total, totalPages } }

## POST /v1/candidates

Create a new candidate.
Required scope: write
Required body fields: first_name, last_name, email
Optional: current_title, location_city, location_country
Response 201: { data: { id, first_name, last_name, email, created_at } }

## GET /v1/placements

List placements for your organisation.
Required scope: read
Query parameters: page, limit, status (default: active)

## Rate Limit Headers

All responses include: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset

## Scopes

| Scope | Endpoints |
|---|---|
| read | GET /v1/candidates, GET /v1/placements |
| write | POST /v1/candidates + all read endpoints |
| admin | All endpoints |
